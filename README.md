# git-lecture-exam

## 개요
- 지난 4일간 학습한 Git/GitHub 사용 능력을 시험하기 위한 Repository입니다.
- 문제는 작업형으로 구성되어 있으며, 각자 본인에게 할당된 부분을 요구사항에 맞게 수정하면 됩니다.
- 기한은 기존 3일에서 1일 단축된 2일 입니다. (**1/25일** 23:59까지)

## 시험 내용
- 시험 코드는 2개의 파일로 구성된 계산기입니다.
  - main.py
    - 프로그램의 실행지점이 포함된 부분입니다.
  - util.py
    - 연산 및 각종 기능이 포함된 부분입니다.
- 위의 코드는 총 8개의 기능 결함이 포함되어 있습니다.
  - 어떠한 결함인지, 어떤 수정이 필요한지 주석을 통해 명시되어 있습니다.
  - 각자 할당된 기능 결함을 수정하시면 됩니다.
- 개인정보 보호를 위해, GitHub Username으로 과제를 할당할 예정입니다.

## 유의사항
- 기능은 반드시 branch를 생성하여 구현하여야 합니다.
- 구현된 기능은 반드시 Pull Request로만 제출해야 합니다.
- 기능 1개당 1회의 commit 이상 수행해야 합니다.
- Merge의 경우, Squash 또는 Rebase를 해서는 안 됩니다.
- **Main에 직접 Push 해서는 안 됩니다**
- commit 메시지는 의미 있게, 어떤 수정을 했는지 작성해야 합니다.
